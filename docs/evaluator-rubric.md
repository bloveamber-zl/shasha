# Evaluator Rubric

用于阶段性检查 agent 输出是否达标。每项 0-2 分：

## 1. 正确性

- 0：目标行为未实现，或实现方向错误。
- 1：主要路径可用，但边界或云端链路不完整。
- 2：目标行为、边界、云端链路都符合需求。

## 2. 验证

- 0：没有运行验证。
- 1：只跑了局部验证，证据不足。
- 2：按功能要求完成验证，并记录证据。

## 3. 范围纪律

- 0：夹带大量无关改动。
- 1：有少量非必要改动，但风险可控。
- 2：改动聚焦当前功能。

## 4. 云开发可靠性

- 0：云函数、数据库或权限规则存在明显风险。
- 1：主要链路可用，但错误处理或权限边界不足。
- 2：鉴权、参数校验、错误返回、权限边界清楚。

## 5. 可维护性

- 0：代码或文档难以接手。
- 1：可接手，但关键约定不够清楚。
- 2：结构清楚，任务状态和证据完整。

## 6. 交接准备度

- 0：下一轮需要重新猜测上下文。
- 1：有部分记录，但缺少下一步或风险说明。
- 2：下一轮可以直接从文档和 `./init.sh` 开始。

## 结论

- `Accept`：总分 10-12，且没有 0 分项。
- `Revise`：总分 7-9，或存在需要补证据的问题。
- `Block`：总分低于 7，或正确性/验证为 0。
